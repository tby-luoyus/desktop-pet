## 1. 引言

### 1.1 目的

本文档为 **DesktopPet** 项目的软件需求规格说明书（SRS），旨在：
- 定义桌面宠物的完整功能需求
- 为系统设计和开发提供依据
- 为验收测试提供评判标准
- 为项目管理和进度追踪提供基准

### 1.2 范围

**DesktopPet** 是一款运行在 Windows 平台的 LLM 驱动桌面宠物应用，核心能力包括：

| 核心能力 | 优先级 | 说明 |
|----------|--------|------|
| LLM对话能力 | P0 | 与用户自然语言对话 |
| 用户状态感知 | P0 | 感知用户当前活动状态 |
| 随机事件驱动 | P1 | 基于时间/状态的主动互动 |
| 视觉表现力 | P1 | 丰富的表情和动画 |
| 现实事件对齐 | P2 | 时间/节日/日程感知 |

### 1.3 定义与术语

| 术语 | 定义 |
|------|------|
| **桌宠** | 桌面宠物的简称，运行在桌面的小动物形象应用 |
| **LLM** | Large Language Model，大语言模型 |
| **感知系统** | 感知用户状态（活动窗口、空闲时间等）的模块 |
| **随机事件** | 基于概率和时间触发的主动性行为 |
| **SRS** | Software Requirements Specification，软件需求规格说明书 |
| **MVP** | Minimum Viable Product，最小可行产品 |
| **Tauri** | 轻量级跨平台桌面应用框架 |
| **Ollama** | 本地大模型运行平台 |

## 2. 总体描述

### 2.1 产品背景

| 项目 | 说明 |
|------|------|
| 产品名称 | DesktopPet |
| 产品定位 | Windows桌面伴侣型AI助手 |
| 目标用户 | 需要桌面陪伴、工作提醒、情感交互的用户 |
| 核心差异 | LLM驱动 + 用户状态感知 + 随机事件 |

### 2.2 产品愿景

> **"你的桌面上的智能小生命——懂你、陪你、提醒你"**

DesktopPet 不只是一个工具，而是一个：
- **陪伴者**: 在你工作时安静待命，在你休息时主动问候
- **助手**: 感知你的状态，在合适的时机提醒你吃饭、休息
- **朋友**: 记住你们的对话，产生情感连接

### 2.3 目标用户画像

```
用户画像: 小明（典型用户）

基本信息:
- 身份: 软件开发者/知识工作者
- 每日电脑使用时长: 8-12小时
- 工作场景: 长时间对着电脑编程、写文档

痛点:
- 沉迷工作忘记吃饭/休息
- 开会时忘记时间
- 工作累了没有人可以简短聊几句
- 需要一个"存在感"但不会打扰的工具

期望:
- 桌面有个可爱的小宠物
- 能在我工作时减少打扰
- 能在我长时间不活动时提醒我
- 能偶尔陪我聊几句

技术接受度:
- 愿意尝试AI对话
- 不希望配置太复杂
- 关注隐私，不希望太多数据被采集
```

### 2.4 约束条件

| 约束类型 | 约束内容 |
|----------|----------|
| **平台约束** | 仅支持 Windows 10/11 (64-bit) |
| **技术约束** | Tauri 2.x + React + TypeScript + Rust |
| **LLM约束** | 支持 OpenAI API / Claude API / Ollama 本地模型 |
| **隐私约束** | 所有感知数据仅本地存储，绝不上传 |
| **性能约束** | 内存占用 < 200MB，后台常驻耗电低 |
| **包体积约束** | 安装包 < 50MB |

### 2.5 假设与依赖

| 假设/依赖 | 说明 |
|-----------|------|
| 用户有网络 | API调用需要网络连接 |
| 用户有LLM API密钥 | 或使用 Ollama 本地模型 |
| Windows 10/11 | 不支持更早的 Windows 版本 |
| 显示器分辨率 >= 1280x720 | 基础分辨率支持 |

### 3.2 功能需求详解

#### UR-001: LLM对话能力 ⭐核心

**需求描述**:
用户可以与桌宠进行自然语言对话，桌宠能够理解上下文、记住重要信息、生成符合人设的回复。

**功能分解**:

| 子功能ID | 子功能名称 | 描述 |
|----------|------------|------|
| UR-001-F1 | 自然对话 | 用户输入文字，桌宠生成回复 |
| UR-001-F2 | 对话上下文 | 记住最近5-10轮对话内容 |
| UR-001-F3 | 长期记忆 | 存储重要对话摘要到本地数据库 |
| UR-001-F4 | 人设一致性 | 回复符合桌宠人设（活泼/温柔等） |
| UR-001-F5 | 多模型支持 | 支持GPT/Claude/Ollama切换 |

**用户故事**:
```
作为一名用户，
我想要和桌宠聊天，
以便在工作疲劳时得到放松。
```

**验收标准**:
- [ ] 用户输入"你好"，桌宠能在3秒内回复
- [ ] 连续5轮对话后，桌宠能记住之前的话题
- [ ] 切换模型后对话质量无明显下降
- [ ] 桌宠的回复不超过50字（单条）

---

#### UR-002: 用户状态感知 ⭐核心

**需求描述**:
桌宠能够感知用户的活动状态（工作中、开会中、空闲中、离开等），并据此调整自己的行为。

**功能分解**:

| 子功能ID | 子功能名称 | 描述 |
|----------|------------|------|
| UR-002-F1 | 活动窗口检测 | 每10秒检测前台窗口 |
| UR-002-F2 | 空闲检测 | 检测用户是否长时间无操作 |
| UR-002-F3 | 状态分类 | 将用户状态分为：工作中/开会中/空闲/离开 |
| UR-002-F4 | 状态响应 | 根据状态调整桌宠行为和对话 |
| UR-002-F5 | 感知频率调节 | 根据用户活动动态调整检测频率 |

**感知频率矩阵**:

| 感知任务 | 检测频率 | 触发条件 |
|----------|----------|----------|
| 窗口活动 | 10秒 | 轮询 |
| 空闲检测 | 30秒 | 轮询 |
| 状态响应 | 事件驱动 | 状态变化时 |
| 屏幕截图 | 5分钟 | 仅用于分析（需授权） |

**用户故事**:
```
作为一名用户，
我希望桌宠能感知我是否在开会，
以便它在开会时保持安静不打扰。
```

**验收标准**:
- [ ] 桌宠能够识别用户正在使用 Office/IDE/浏览器
- [ ] 用户离开5分钟后，桌宠进入睡眠动画
- [ ] 用户工作繁忙时，桌宠主动打扰频率降低50%
- [ ] 感知功能可在设置中完全关闭

---

#### UR-003: 时间感知行为 ⭐核心

**需求描述**:
桌宠能够感知当前时间，在不同时间段展现不同行为和对话。

**功能分解**:

| 子功能ID | 子功能名称 | 描述 |
|----------|------------|------|
| UR-003-F1 | 时段问候 | 早安/午安/晚安等时段问候 |
| UR-003-F2 | 餐食提醒 | 饭点提醒用户用餐 |
| UR-003-F3 | 节日祝福 | 传统节日主动祝福 |
| UR-003-F4 | 天气关联 | 根据天气给予问候（可选） |

**时间行为映射表**:

| 时间段 | 行为模式 | 示例对话 |
|--------|----------|----------|
| 06:00-09:00 | 活力模式 | "早上好！今天有什么计划吗？" |
| 09:00-12:00 | 工作模式 | "工作加油！需要我帮忙查资料吗？" |
| 12:00-14:00 | 午餐模式 | "该吃午饭啦，不要饿着肚子哦~" |
| 14:00-18:00 | 下午模式 | "下午茶时间要不要休息一下？" |
| 18:00-22:00 | 晚间模式 | "辛苦了一天，晚饭吃什么？" |
| 22:00-06:00 | 睡眠模式 | "晚安，我先睡啦~" |

**用户故事**:
```
作为一名用户，
我希望桌宠能在饭点提醒我吃饭，
以便我养成良好的饮食习惯。
```

**验收标准**:
- [ ] 12:00-13:00 之间触发午餐提醒（可关闭）
- [ ] 18:00-19:00 之间触发晚餐提醒（可关闭）
- [ ] 22:00后桌宠进入睡眠状态（视觉变化）
- [ ] 节日当天0点自动更新节日祝福

---

#### UR-004: 随机事件系统

**需求描述**:
桌宠会随机触发一些主动性行为，增加趣味性和不可预测性。

**功能分解**:

| 子功能ID | 子功能名称 | 描述 |
|----------|------------|------|
| UR-004-F1 | 随机对话 | 随机时间主动发起对话 |
| UR-004-F2 | 随机动作 | 随机播放动画（伸懒腰、打哈欠等） |
| UR-004-F3 | 事件种子 | 基于日期+用户ID生成确定性随机 |
| UR-004-F4 | 疲劳衰减 | 随机事件频率随连续打扰递减 |

**随机事件类型**:

| 事件类型 | 触发概率 | 触发间隔 | 持续时间 |
|----------|----------|----------|----------|
| 随机问候 | 10%/小时 | 至少30分钟 | 即时 |
| 趣味问题 | 5%/小时 | 至少1小时 | 即时 |
| 卖萌动作 | 20%/小时 | 至少10分钟 | 3-5秒 |
| 发呆动作 | 15%/小时 | 至少15分钟 | 5-10秒 |

**用户故事**:
```
作为一名用户，
我希望桌宠偶尔会主动来找我聊天，
让我感觉它是真的在"陪"我。
```

**验收标准**:
- [ ] 平均每小时触发1-3次随机事件
- [ ] 连续打扰3次后，事件触发概率降低
- [ ] 用户长时间无互动后，随机事件概率恢复正常
- [ ] 随机事件可通过设置关闭

---

#### UR-005: 视觉动画表现

**需求描述**:
桌宠拥有丰富的视觉动画，包括待机、情绪、动作等状态。

**功能分解**:

| 子功能ID | 子功能名称 | 描述 |
|----------|------------|------|
| UR-005-F1 | 待机动画 | 基础待机呼吸动画 |
| UR-005-F2 | 情绪动画 | 开心/难过/好奇/困倦等表情 |
| UR-005-F3 | 动作动画 | 行走/跳跃/睡觉/吃东西 |
| UR-005-F4 | 气泡对话 | 对话内容以气泡形式展示 |
| UR-005-F5 | 拖拽交互 | 用户可拖拽桌宠位置 |

**动画状态机**:

```
                    ┌─────────────┐
                    │   IDLE      │ ← 默认待机
                    └──────┬──────┘
                           │
          ┌────────────────┼────────────────┐
          │                │                │
          ▼                ▼                ▼
    ┌──────────┐     ┌──────────┐     ┌──────────┐
    │  HAPPY   │     │  SAD     │     │ CURIOUS  │
    │  (开心)  │     │  (难过)  │     │  (好奇)  │
    └──────────┘     └──────────┘     └──────────┘
          │                │                │
          └────────────────┴────────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  SLEEPING   │ ← 睡眠模式
                    └─────────────┘
```

**资源要求**:

| 动画类型 | 帧率 | 帧数 | 尺寸建议 |
|----------|------|------|----------|
| 待机呼吸 | 8fps | 4-8帧 | 128x128 |
| 行走 | 12fps | 6-8帧 | 128x128 |
| 跳跃 | 12fps | 8-12帧 | 128x128 |
| 睡眠 | 4fps | 4帧 | 128x128 |
| 情绪表情 | 8fps | 2-4帧 | 局部叠加 |

**用户故事**:
```
作为一名用户，
我希望桌宠有可爱的动画，
让我在使用电脑时感到放松和愉悦。
```

**验收标准**:
- [ ] 桌宠有至少3种情绪状态动画
- [ ] 对话以气泡形式展示
- [ ] 桌宠可被拖拽到屏幕任意位置
- [ ] 动画流畅度 >= 24fps

| **State Layer** | 管理应用状态 | 提供get/set接口 |
| **LLMGateway** | 统一LLM调用接口 | 支持多模型自动降级 |
| **EventEngine** | 随机事件调度 | 时间触发、状态触发 |
| **PerceptionManager** | 用户状态感知 | Win32 API调用 |
| **MemoryManager** | 对话历史+长期记忆 | SQLite本地存储 |
| **Platform Layer** | 系统集成 | Tauri Command |

### 6.1 性能指标

| 指标 | 要求 | 测量方法 |
|------|------|----------|
| **启动时间** | < 3秒 | 冷启动到完全可交互 |
| **内存占用** | < 200MB | 正常运行状态 |

| **安装包大小** | < 50MB | 打包后安装包 |

### 6.2 性能优化策略

```
优化策略:

1. 感知系统节能
   └── 空闲时降低检测频率
   └── 使用事件驱动代替轮询

2. LLM调用优化
   └── 启用流式输出
   └── 实现请求缓存

3. 动画性能
   └── 使用CSS动画代替JS动画
   └── 必要时降低帧率
```

## 7. 安全与隐私需求

### 7.1 隐私保护原则

```
┌─────────────────────────────────────────────────────────────┐
│                     隐私保护五原则                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 数据最小化                                               │
│     └── 仅收集功能必需的数据                                   │
│                                                              │
│  2. 本地优先                                                 │
│     └── 所有数据存储在本地，绝不主动上传                        │
│                                                              │
│  3. 用户可控                                                 │
│     └── 所有感知功能可随时关闭                                  │
│                                                              │
│  4. 透明可查                                                 │
│     └── 用户可查看被采集的数据内容                              │
│                                                              │
│  5. 安全存储                                                 │
│     └── 敏感数据（API Key）加密存储                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 感知数据清单

| 数据类型 | 采集方式 | 存储位置 | 是否上传 |
|----------|----------|----------|----------|
| 活动窗口名称 | Win32 API | 本地SQLite | ❌绝不 |
| 用户空闲时长 | Win32 API | 本地SQLite | ❌绝不 |
| 对话内容 | 用户输入 | 本地SQLite | ❌绝不 |
| 系统时间 | 系统API | 不存储 | N/A |
| API Key | 用户输入 | 加密存储 | ❌绝不 |

### 7.3 感知范围边界

**✅ 允许感知**:
- 前台窗口的应用包名/进程名
- 窗口标题文本
- 系统时间
- 用户空闲/活动状态

**❌ 严禁感知**:
- 窗口内部内容（文档、聊天、邮件）
- 键盘输入内容
- 文件系统内容
- 麦克风音频内容（除非用户主动开启语音）
- 屏幕截图内容（除非用户主动分享）

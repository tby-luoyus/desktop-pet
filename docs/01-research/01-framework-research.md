# Windows桌面宠物技术选型调研报告

## 一、Windows桌面应用框架对比

### 1. Electron

| 评估维度 | 详情 |
|---------|------|
| **开发语言生态** | JavaScript/TypeScript + Node.js，前端开发者友好 |
| **包体积大小** | **150-300MB**（包含完整Chromium引擎） |
| **系统资源占用** | 内存占用高，空载约 **300-500MB**，CPU占用明显 |
| **LLM集成友好度** | **极高** - npm生态丰富，OpenAI/Anthropic SDK完善 |
| **2D动画/图形渲染支持** | 依赖前端框架（React/Vue）+ CSS动画，或Canvas/pixi.js |
| **打包分发便捷度** | 成熟，electron-builder一键打包 |
| **开源许可证** | MIT License（核心开源，但需遵守Chromium许可） |

**代表应用**: VS Code、Slack、Discord、Obsidian

---

### 2. Tauri

| 评估维度 | 详情 |
|---------|------|
| **开发语言生态** | Rust（后端）+ 前端任意框架（React/Vue/Svelte等） |
| **包体积大小** | **5-20MB**（利用系统原生WebView） |
| **系统资源占用** | 内存占用极低，空载约 **30-80MB** |
| **LLM集成友好度** | **高** - HTTP调用LLM API，前端JS库实现流式输出 |
| **2D动画/图形渲染支持** | 依赖前端框架，Web技术栈实现动画效果 |
| **打包分发便捷度** | 成熟，Tauri CLI一键打包 |
| **开源许可证** | MIT License + Apache-2.0 |

**代表应用**: Hexchat、Sparky

> Tauri官方文档（v2.0）指出核心优势："利用系统原生WebView，实现更小打包体积"

---

### 3. Unity

| 评估维度 | 详情 |
|---------|------|
| **开发语言生态** | C#为主 |
| **包体积大小** | **100-500MB+**（IL2CPP打包后较大） |
| **系统资源占用** | 内存占用中等偏高，空载约 **200-400MB** |
| **LLM集成友好度** | 中等 - HTTP请求调用LLM API |
| **2D动画/图形渲染支持** | **优秀** - 原生2D Pipeline，Animation系统完善 |
| **打包分发便捷度** | 成熟，Unity Build系统支持多平台 |
| **开源许可证** | Unity Personal（有营收限制）/ Unity Pro（订阅制）|

> **注意**: 2026年3月Unity已终止向中国大陆及港澳地区提供海外资源商店访问权限

---

### 4. Godot

| 评估维度 | 详情 |
|---------|------|
| **开发语言生态** | GDScript（类Python）、C#、C++可选 |
| **包体积大小** | **30-100MB** |
| **系统资源占用** | 内存占用较低，空载约 **100-200MB** |
| **LLM集成友好度** | 中等 - HTTP请求调用LLM API |
| **2D动画/图形渲染支持** | **优秀** - Godot 4原生2D引擎 |
| **打包分发便捷度** | 较成熟，Export Templates支持多平台 |
| **开源许可证** | **MIT License**（完全开源免费） |

---

### 5. 自研框架（WPF/WinUI）

| 评估维度 | 详情 |
|---------|------|
| **开发语言生态** | C#/.NET |
| **包体积大小** | **10-50MB** |
| **系统资源占用** | 低，直接调用Windows API |
| **LLM集成友好度** | 高 - .NET HttpClient调用LLM API方便 |
| **2D动画/图形渲染支持** | 中等 - WPF XAML动画 |
| **打包分发便捷度** | 一般 - 需处理.NET运行时依赖 |
| **开源许可证** | MIT/Apache-2.0（.NET） |

## 三、针对LLM对话桌宠的推荐方案

### 最佳推荐: Tauri + React/Vue

**推荐理由**:

1. **包体积最小化**: 5-20MB vs Electron的150MB+
2. **资源占用低**: 空载30-80MB，可常驻后台
3. **前端生态丰富**: React/Vue + CSS动画实现流畅2D宠物
4. **LLM集成便利**: Rust HTTP Client调用API，前端流式渲染
5. **安全可靠**: Rust内存安全，长时间运行稳定
6. **开源透明**: MIT许可，无合规风险

**架构示意**:

`
┌─────────────────────────────────┐
│         Tauri 桌面应用           │
│  ┌───────────┐  ┌────────────┐ │
│  │ React/Vue │  │ Rust后端   │ │
│  │ 前端UI层   │◄─┤ HTTP调用   │ │
│  │ 动画渲染   │  │ LLM API    │ │
│  └───────────┘  └────────────┘ │
└─────────────────────────────────┘
`

### 备选方案

| 场景 | 推荐 | 理由 |
|------|------|------|
| 预算有限/2D为主 | Godot | MIT免费，2D原生支持好 |
| 高度复杂游戏逻辑 | Unity | 动画系统最成熟，需评估许可 |
| 已有WPF经验 | WPF + .NET | 内部项目快速开发 |

---

## 四、结论

**Tauri + 前端框架（React/Vue/Svelte）** 是LLM驱动桌面宠物的最优技术选型：

- 开发效率高，前端团队可快速上手
- 用户体验佳，极小包体积+低内存占用
- LLM集成便利，架构清晰，流式输出易实现
- 商业风险低，MIT许可，无营收限制

如以游戏化动画为主可考虑Godot；如需高度复杂游戏逻辑可考虑Unity（需评估许可政策）。
